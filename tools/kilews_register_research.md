# Kilews KL-NTCS-M7 MODBUS 参数写入地址研究

## 设备信息

- 型号：奇力速 KL-NTCS-M7 拧紧控制器
- 连接：MODBUS TCP，192.168.0.105:502，Unit ID=1
- 工控机：192.168.0.99，Windows，通过 paramiko SSH 部署

## 背景

目标是找到通过 MODBUS 修改拧紧参数（目标扭矩、目标角度、转速、上下限等）的正确寄存器地址。说明书给出的地址写入后回读正确、控制器重启后仍保留，但**不影响实际拧紧行为**。

## 已知的寄存器布局

### 控制寄存器区 (460-464) — 可读写，功能正常

| 地址 | 功能 | 验证 |
|------|------|:--:|
| 461 | 工具使能 (1=启用, 0=禁用) | ✅ 有效 |
| 462 | 控制器重启 (写1) | ✅ 有效 |
| 463 | JOB 选择 | ✅ 有效，切换后4166+区参数不变 |
| 464 | SEQ 选择 | ✅ 有效 |

### 拧紧结果区 (4155-4165) — 只读，每拧紧一次更新

| 地址 | 含义 | 实测值示例 |
|------|------|-----------|
| 4156 | 拧紧结果扭矩 (raw) | 816→0.816 N·m, 2730→2.730 N·m |
| 4158 | 拧紧状态码 | 510, 1108 |
| 4160 | 拧紧结果角度 (raw) | 2986→2986°, 903→903° |
| 4162-4164 | 其他结果标志 | 1, 1, 5 |

### 工作参数区 (4166-4200) — MODBUS 只读，物理按键可写

这是控制器**实际使用的参数**，通过物理按键修改后变化，但 MODBUS 写入返回 ACK 而值不变（静默忽略）。

| 地址 | 32-bit | 按键改前 | 按键改后(2.5Nm目标) | 含义推测 |
|------|--------|----------|---------------------|----------|
| 4166-4167 | u32 | 7405568 | 8650752 | 序列号/计数器 |
| 4168-4169 | u32 | 66036 | 66036 | JOB=1(hi) + 速度=500(lo) |
| 4170-4171 | u32 | **700** | **2500** | **目标扭矩** (÷1000=N·m) |
| 4172-4173 | u32 | 7700 | 3000 | 角度相关参数 |
| 4174-4175 | u32 | 0 | 2000 | 参数4 |
| 4178-4179 | u32 | 30600 | 1200 | 参数8 |
| 4190-4191 | u32 | 2986 | 903 | 上次拧紧角度结果 |

**关键证据**：用户通过物理按键将目标扭矩从 0.7 N·m 改为 2.5 N·m 后：
- 4171 从 700 → 2500 ✓
- 实际拧紧结果 2.73 N·m（接近目标 2.5 N·m）✓

### 参数缓冲区 (1144-1163) — MODBUS 可读写，但无效

说明书标注的 STEP1 参数地址。写入成功、回读正确、重启后保留，但**控制器不使用这些值**。

| 地址 | 说明书标注 | 写入测试 |
|------|-----------|:--:|
| 1144 | 目标类型 (2=扭力, 1=角度) | ✅ 可写 |
| 1145-1146 | 目标角度 (32-bit) | ✅ 可写 |
| 1147-1148 | 目标扭力 (32-bit) | ✅ 可写 |
| 1151 | 转速 | ✅ 可写 |
| 1155-1156 | 扭力上限 (32-bit) | ✅ 可写 |
| 1157-1158 | 扭力下限 (32-bit) | ✅ 可写 |
| 1160-1161 | 角度上限 (32-bit) | ✅ 可写 |
| 1162-1163 | 角度下限 (32-bit) | ✅ 可写 |

**关键证据**：用户通过物理按键改参数后，1146 从 900→3000，1148 从 4550→700——缓冲区被控制器覆写为按键设置的值。说明 1144-1163 可能是 EEPROM 的**只读映射窗口**（从控制器→MODBUS 方向同步），而非写入入口。

### 指针/描述符区 (1100-1143) — 部分只读

```
1100-1102: [50, 1200, 1]  ← 头部
1107-1112: [1, 1, 1, 1, 1, 1]  ← 全部为1，只读
1120-1124: [1, 1, 1, 1, 1]  ← 全部为1，只读
1135: 1  ← 步骤使能
1136-1143: [1108, 1109, 1110, 1111, 1112, 1120, 1121, 1122]  ← 指针表！值恰好是1108-1122的地址
```

1136-1143 是指针表（值恰好等于 1108-1122 的地址），但指向的地址（1108-1124）全部为 0 或 1，也是只读。

## 尝试过的同步/生效方法（全部失败）

以下操作均在写入 1144-1163 缓冲区后执行，检查 4170-4171 是否变化，**无一成功**：

1. 463=1（JOB 选择生效）
2. 463=0→463=1（JOB 切换）
3. 463=2→463=1（JOB=2 再切回 JOB=1）
4. 464=0→464=1（SEQ 切换）
5. 1135=0→1135=1（步骤关闭再开启）
6. 461=0→461=1（工具禁用再启用）
7. 462=1（控制器重启）
8. 463=0→写参数→463=1（取消 JOB→写→重选 JOB）
9. 1135=0→写参数→1135=1
10. 0x10 批量写入 1144-1163（20个寄存器一次性写入）
11. 0x06 直接写入 4171（MODBUS ACK 但值不变）
12. 0x10 批量写入 4170-4173（MODBUS ACK 但值不变）
13. 禁用状态(461=0)下写入 4171（ACK 但值不变）
14. 扫描 0-200 范围找"写使能"寄存器，逐一置1后测试 4171 可写性——未找到

## 扫描覆盖范围

已扫描以下地址范围的非零寄存器：
- 0-500（控制区）
- 1100-1200（参数缓冲区+指针表）
- 1200-2000（仅 1284/1409/1534/1659=2）
- 2000-3000（仅 2135-2136 非零）
- 3000-4000（全零）
- 4000-4300（部分非零，含拧紧结果和工作参数）
- 5000-9000（全零）
- 9000-50000（全零）

**结论：除 1144-1163 外，未发现任何可写入且能影响 4166+ 或拧紧行为的寄存器。**

## 扭矩单位换算

从寄存器 264 读取：
- 0=kgf·m (×10000)
- 1=N·m (×1000)
- 2=kgf·cm (×100)
- 3=lbf·in (×100)
- 4=cN·m (×10)

当前设备 unit=1（N·m），multiplier=1000。

## 核心疑问

1. 物理按键修改参数时，MODBUS 寄存器空间是否出现变化？——已验证：4166+ 区会变，但该区 MODBUS 只读。
2. 是否存在不同于 1144-1163 的参数写入地址？
3. 是否需要特定的 MODBUS 功能码或写入序列（如先写密码/授权寄存器）？
4. 说明书的寄存器表是否对应不同固件版本或不同型号？
5. 奇力速是否有专用配置软件？该软件通过什么协议/地址修改参数？
