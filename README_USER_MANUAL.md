## 一、系统概述

膨胀阀拧紧防错追溯 HMI 是一套基于 Windows 的工业自动化上位机系统，用于膨胀阀装配线的视觉检测、拧紧控制与产品追溯。

**核心功能**：
- 海康工业相机实时采集图像，YOLO 视觉模型检测 O 型圈与膨胀阀
- 与西门子 S7-200 SMART PLC 通信，实现 7 步自动化流程握手
- 控制奇力速 KL-NTCS-M7 拧紧控制器，实时监控扭矩与角度
- 扫码枪自动触发，二维码绑定产品追溯
- 生产记录自动保存，支持按日期/关键字/状态查询与 Excel 导出
- 拧紧数据防错：NG 结果不记录，强制重新拧紧直至合格

**访问地址**：`http://192.168.0.99:8010`（仅限工控机本机或同网段设备访问）

## 三、安装与启动

### 3.1 软件环境

- Python 3.12 或 3.14（安装在 `C:\Python314\`）
- Windows 10/11 64 位
- 海康 MVS SDK（安装在 `C:\Program Files (x86)\MVS\`）

### 3.2 项目目录

项目部署在 `D:\expansion_valve_hmi\`，生产数据保存在 `D:\膨胀阀装配数据\`。

### 3.3 启动系统

双击 `D:\expansion_valve_hmi\启动HMI.bat`，自动完成以下操作：

1. 强制停止已有的 Python 进程
2. 最小化启动 HMI 后端服务（端口 8010）
3. 等待服务就绪后自动打开浏览器

如需手动启动，运行：
```
C:\Python314\python.exe D:\expansion_valve_hmi\run.py --host 0.0.0.0 --port 8010
```

### 3.4 停止系统

直接关闭浏览器窗口不会停止后端。如需停止所有服务，按 `Ctrl+C` 或运行：
```
taskkill /F /IM python.exe
```

## 五、生产操作流程

### 5.1 准备工作

1. 确认所有硬件已通电（相机、PLC、拧紧枪、扫码枪）
2. 双击 `启动HMI.bat`，等待浏览器自动打开
3. 在生产页面选择正确的 **产品型号** 和 **操作员**
4. 确认顶部状态栏显示 `PLC: Online`、`扫码枪: COM3 已连接`
5. 如果 PLC 显示手动模式（M0.3=1），自动流程不会启动，需将 PLC 切到自动模式

### 5.2 自动流程 7 步详解

#### 步骤 1 — O 型圈检测 (vision_wait_stable)

- 操作员将产品放置到相机视野内
- 系统自动检测 O 型圈数量和稳定性
- 条件：检测到 **2 个 O 型圈** 且画面 **静止 ≥ 2 秒**
- 通过后自动抓拍一张图片，进入下一步

#### 步骤 2 — 膨胀阀检测 (vision_check_cover)

- 操作员将膨胀阀放到 O 型圈上方
- 系统使用 YOLO 视觉模型检测 TXV（膨胀阀）或 QR 码
- 条件：检测到 **TXV 或 QR 目标**，且覆盖率达到阈值
- 通过后立即发送 **PLC M0.0 = 1**（产品就绪），进入下一步

#### 步骤 3 — 压紧 (plc_handshake)

- 系统等待 PLC 执行压紧动作
- 系统已发送 M0.0，PLC 应控制压紧机构夹紧
- 条件：PLC 返回 **M0.5 = 1**（PLC 就绪，持续 1 秒脉冲）
- 通过后系统清空拧紧基线数据，进入拧紧等待

#### 步骤 4 — 拧紧 (tightening_wait)

- 操作员使用奇力速拧紧枪依次拧紧两颗螺丝
- 系统每 500ms 轮询一次拧紧控制器 MODBUS 寄存器
- 每次拧紧完成后自动读取扭矩和角度数据
- **NG 处理**：如果某颗螺丝拧紧结果为 NG，数据**不记录**，该螺丝保持 "WAIT" 状态，操作员需重新拧紧直到 OK
- **超时处理**：超过拧紧超时时间无结果，视为超时

#### 步骤 5 — 判定 (tightening_eval)

- 条件：两颗螺丝全部拧紧完成且结果均为 OK
- 发送 **PLC M0.1 = 1**（拧紧完成且合格）
- 分配正式序列号，记录拧紧完成时间
- 自动触发扫码枪
- 如果两颗螺丝中有 NG，发送 **PLC M1.0 = 1**，进入 NG 等待返修流程

#### 步骤 6 — 扫码 (pending_scan)

- 拧紧合格后扫码枪自动触发（发送 HEX 指令 `16 54 0D`）
- 操作员将产品二维码对准扫码枪
- 系统校验二维码格式（默认正则规则），检查是否重复
- 扫码成功 → 绑定到记录 → 发送 **PLC M0.2 = 1**
- 如不需要扫码绑定，可关闭产品配方中的 `enable_qr_binding` 开关

#### 步骤 7 — 完成 (complete)

- 当前周期全部完成
- 系统自动重置状态，准备下一轮生产
- 下一轮自动从步骤 1 开始（需 M0.3 = 0 即自动模式）

### 5.3 手动操作

#### 扫码
在 `pending_scan` 状态下可手动输入 QR 码，点击「绑定二维码」按钮。

#### 跳过扫码
点击「跳过扫码」按钮，直接完成当前周期。

#### 返修
拧紧结果 NG 时，系统进入 `ng_wait_rework` 状态。PLC 侧选择返修/放行后，点击「返修后扫码」进行返修记录。

#### PLC 复位
PLC 发送 **M0.6 = 1** 触发复位，当前记录标记为 RESET，系统回到初始状态。

## 六、PLC 信号对照表

### 6.1 PC → PLC 输出（PC 写 M 位）

| M 位 | 含义 | 触发条件 |
|------|------|----------|
| M0.0 | 产品就绪 | O 型圈 + 膨胀阀均检测通过 |
| M0.1 | 拧紧完成 (OK) | 两颗螺丝全部 OK |
| M0.2 | 扫码完成 | 二维码绑定成功 |
| M0.7 | 禁止扫码 | 已跳过扫码 |
| M1.0 | 拧紧 NG | 至少一颗螺丝 NG |

### 6.2 PLC → PC 输入（PLC 写 M 位）

| M 位 | 含义 | 说明 |
|------|------|------|
| M0.3 | 手动模式 | =1 时自动流程不启动 |
| M0.4 | 急停 | =1 时禁止操作 |
| M0.5 | PLC 就绪 | 压紧完成后 PLC 置 1（脉冲） |
| M0.6 | 复位 | PLC 复位当前流程 |

### 6.3 PLC 状态只读

- M10.2：拧紧完成（PLC 侧）
- PLC 心跳计数持续递增
- PLC 告警码（0 = 无告警）

## 十、数据记录规则

### 10.1 序列号格式

```
产品型号-日期-工位号-序号
```

示例：`HB11-20260521-ST01-0001`

### 10.2 序号规则

- 每天每种产品型号的序号从 **0001** 开始自动递增
- 仅当两颗螺丝全部拧紧为 OK 时，才分配正式序号
- 中途复位（M0.6）或拧紧超时的周期**不消耗**序号
- 序号的日期为拧紧完成当天的日期

### 10.3 记录时间

- 「时间」列为两颗螺丝全部拧紧完成的时刻（精确到秒）
- 序号中日期为拧紧完成当天的日期

### 10.4 NG 处理

- 拧紧结果 NG 时，数据不写入螺丝记录
- 该螺丝保持 "WAIT" 状态
- 操作员必须重新拧紧直到 OK
- 如果持续无法达到 OK，可通过 PLC 复位（M0.6）放弃当前周期

### 10.5 图片保存

- 每次 O 型圈检测通过时自动抓拍
- 图片右下角叠加时间戳水印
- 保存路径：`D:\膨胀阀装配数据\images\{产品型号}\{日期}\{时间戳}_{型号}_NOQR.jpg`

## 十三、故障排除

### 13.1 网页打不开

1. 确认工控机已开机，HMI 已启动
2. 检查 Python 进程：`tasklist | findstr python`
3. 检查端口：`netstat -an | findstr 8010`
4. 如果 HMI 未运行，双击 `启动HMI.bat`
5. 如果端口被占用，先执行 `taskkill /F /IM python.exe`，再启动

### 13.2 相机无画面

1. 确认相机已通电（网口指示灯亮）
2. 确认相机 IP 为 `192.168.0.101`
3. 确认 Windows 防火墙未阻止相机
4. 检查海康 MVS 客户端能否搜到相机
5. 如果在 MVS 中能搜到但 HMI 中无画面，检查 `mvs_path` 设置

### 13.3 PLC 未连接

1. 确认 PLC 已通电
2. 确认 PLC IP 为 `192.168.0.10`
3. 确认工控机能 ping 通 PLC：`ping 192.168.0.10`
4. 检查 PLC 设置中的 Rack/Slot 是否正确
5. PLC 为 S7-200 SMART，Rack=0, Slot=1

### 13.4 拧紧数据不显示

1. 确认拧紧控制器已通电
2. 确认 IP 为 `192.168.0.105`，端口 `502`
3. 在设置中点击「连接控制器」
4. 如果已连接，检查「奇力速拧紧」面板的状态显示
5. 确认产品参数已写入（切换产品型号会自动写入）

### 13.5 扫码枪不工作

1. 确认扫码枪已连接 COM3
2. 在设置中确认扫码枪模式为 `serial`，端口为 `COM3`
3. 观察页面顶部状态栏扫码枪状态
4. 手动扫码测试：用扫码枪扫描任意条码，观察「扫描记录」是否显示
5. 检查设备管理器中 COM3 是否正常，无黄色感叹号

### 13.6 自动流程不启动

1. 确认 PLC 不在手动模式（M0.3 ≠ 1）
2. 确认 PLC 未急停（M0.4 ≠ 1）
3. 确认产品型号和操作员已选择
4. 确认相机 YOLO 推理正常运行（画面中有检测框）
5. 确认自动化开关在设置中已启用

### 13.7 查看运行日志

```
D:\expansion_valve_hmi\runtime\hmi_stdout.log  （标准输出）
D:\expansion_valve_hmi\runtime\hmi_stderr.log  （错误输出）
```

日志中包含相机、PLC、拧紧、扫码的详细运行信息。

## 十四、日常维护

### 14.1 每日

- 启动后确认相机画面、PLC 连接、拧紧枪连接正常
- 检查当日记录数量是否正常递增
- 确认扫码枪能正常读取
