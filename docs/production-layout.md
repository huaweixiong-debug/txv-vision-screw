# 生产页布局重构说明

这份文档不是记录现状，而是给 Claude Code 的直接改版说明。  
目标只有一个：**把生产页改成更稳、更整洁、更像工业 HMI，而不是一堆浮动卡片堆在一起。**

## 范围

- 只改布局、视觉层级、间距、字号、卡片比例。
- **不要改通讯、PLC、扫码、相机、接口逻辑。**
- 可以修改的文件：
  - `web/index.html`
  - `web/styles.css`
  - `web/app.js`

## 当前问题

当前页面的问题不是某一张卡片，而是整体秩序感不够：

1. 面板太多、样式都很抢，主次不清。
2. 顶部摘要、左侧流程、中间视觉、右侧记录都在争夺注意力。
3. 卡片圆角、阴影、渐变都偏重，导致工业页面看起来发飘。
4. 左列三张卡片高度关系杂乱，中心视觉区和右侧记录区没有形成稳定对齐。
5. O 型圈视觉虽然是核心，但周围 KPI、小按钮、小提示过多，视觉焦点被削弱。
6. 最近记录区域太像普通表格，缺少“右侧稳定信息栏”的感觉。

## 目标效果

改完之后，页面应该有这几个感受：

- 第一眼先看到 **当前步骤 + O 型圈视觉结果**。
- 第二眼看到 **最近记录**。
- 第三眼才看左侧的人员、PLC、扫码辅助区。
- 整体更像“设备控制台”，不是“后台管理页”。
- 信息密度高，但不乱；能一眼扫到重点。

## 信息层级

页面层级固定为三层：

1. **顶部状态层**
   - 只保留最关键状态：当前步骤、序列号/二维码、产品/配方。
   - 顶部只做摘要，不要承担详细操作。

2. **主体工作层**
   - 中间：O 型圈视觉，必须成为绝对主角。
   - 右侧：最近记录，作为稳定的纵向信息栏。
   - 左侧：操作员、PLC、扫码，作为辅助控制列。

3. **底部执行层**
   - 奇力速拧紧作为和视觉强相关的执行结果，放在视觉区下方，宽度对齐。

## 必须采用的版式

### 1. 顶部摘要区改成更克制的一排

位置：`#production .production-summary`

要求：

- 维持三块，但弱化右边两张 `metric-card` 的存在感。
- `hero-card` 保留深色，但宽度更明确，承担“当前步骤 + 报警 + 流程状态”。
- 两张 `metric-card` 改成更安静的浅色卡，只显示核心数据，不要有很重的装饰。
- `automation-bar` 视觉上并入顶部摘要区，像一条细工具条，不要再像独立大卡片。

建议宽度关系：

- `hero-card`: 1.55
- `serial/qr`: 0.9
- `product/recipe`: 0.95

### 2. 主体区域固定为三栏秩序

不要让页面像自由散落。保留拖拽能力，但默认布局必须先是整齐的。

建议默认布局：

```js
const DEFAULT_LAYOUT = {
  "operator-panel":   { x: 0.00, y: 0.00, w: 0.21, h: 0.24 },
  "plc-panel":        { x: 0.00, y: 0.25, w: 0.21, h: 0.33 },
  "scan-panel":       { x: 0.00, y: 0.59, w: 0.21, h: 0.17 },
  "camera-panel":     { x: 0.22, y: 0.00, w: 0.44, h: 0.60 },
  "tightening-panel": { x: 0.22, y: 0.61, w: 0.44, h: 0.18 },
  "recent-panel":     { x: 0.67, y: 0.00, w: 0.33, h: 0.79 },
};
```

布局原则：

- 左列窄且稳定，只放辅助区。
- 中列最大，给 O 型圈视觉。
- 右列完整给最近记录，不要碎。
- 底部拧紧面板和视觉面板左对齐、同宽，看起来像一个工作区上下两层。

### 3. O 型圈视觉卡片必须成为页面主角

位置：`.camera-panel`

要求：

- 卡片面积最大。
- 图片区域优先，右侧信息区次之。
- 识别数量必须最大号，位置要稳定。
- “置信度 / 稳定度 / 覆盖率”不要都做成同等权重的卡片，否则又乱。

建议结构：

- 左边 70%：实时图像
- 右边 30%：结果摘要
- 右侧仅保留三类信息：
  - 识别数量
  - 当前判定
  - 一行辅助状态

建议做法：

- `vision-kpis` 从四个小盒子收敛成两个主块 + 一个简短状态行。
- `vision-message` 改成结果摘要卡，不要像说明文。
- 底部进度条保留，但做细，弱化存在感。
- 手动测试按钮放到底部一行，尺寸收紧，不要抢视觉。

### 4. 左侧三张卡片做成同一列工具区

位置：

- `.operator-panel`
- `.plc-panel`
- `.scan-panel`

要求：

- 三张卡片视觉语气一致。
- 标题更小，内边距更紧，按钮高度统一。
- 不要每张卡片都像一个“主模块”，它们应当是辅助面板。

具体要求：

- `operator-panel` 更紧凑，突出“人员 + 产品 + 开始生产”。
- `plc-panel` 去掉过强的块状感，复选项更规整，像状态清单。
- `scan-panel` 只保留必要输入和按钮，历史记录弱化。

### 5. 最近记录改成右侧稳定栏

位置：`.recent-panel`

要求：

- 高度贯穿主体区域，形成稳定的右侧边栏。
- 表头固定，表格内容清楚，配色克制。
- 不要再让右上角按钮太大。

建议：

- `panel-head` 更扁平，标题居中，刷新按钮改成小号次按钮。
- 表格字体略小于主区，但行高更稳定。
- 状态标签统一成一套小胶囊，减少颜色数量。
- 右栏整体比中心视觉更安静，不能喧宾夺主。

## 视觉风格要求

### 1. 降低“漂浮感”

Claude Code 需要主动收敛以下样式：

- 圆角减小一档
- 阴影减轻
- 玻璃模糊效果减轻
- 非核心卡片去掉大面积渐变

建议方向：

- 主卡片圆角：`16px - 18px`
- 次卡片圆角：`12px - 14px`
- 阴影只保留一层轻阴影
- 只有 `hero-card` 和视觉结果主块允许更强对比

### 2. 间距统一

不要每个区块都有自己的节奏，统一成一套：

- 面板外间距：`8px`
- 面板内边距：`12px - 14px`
- 标题到底部内容：`8px`
- 小组件间距：`6px - 8px`

### 3. 标题层级统一

要求：

- 面板标题字重高，但字号不要过大。
- 数值比标题大一档。
- 说明文字全部降到辅助色。
- 不要让每个文字块都像标题。

建议：

- 面板标题：`14px - 16px`
- 普通字段：`12px - 13px`
- 关键数字：`28px - 40px`
- 辅助说明：`11px - 12px`

## 需要直接修改的代码点

### `web/app.js`

1. 修改 `DEFAULT_LAYOUT`
2. 把

```js
const LAYOUT_STORAGE_KEY = "hmi_production_layout_v2";
```

改成新版本，例如：

```js
const LAYOUT_STORAGE_KEY = "hmi_production_layout_v3";
```

目的：

- 强制让浏览器丢掉旧布局缓存
- 让新的默认布局直接生效

### `web/styles.css`

重点重构这些区域：

- `.production-summary`
- `.hero-grid`
- `.production-board .panel`
- `.hero-card`
- `.metric-card`
- `.automation-bar`
- `.camera-panel`
- `.vision-stage`
- `.vision-kpis`
- `.vision-message`
- `.operator-panel`
- `.plc-grid`
- `.scan-panel`
- `.recent-panel`
- `.recent-panel .table-wrap`
- `.status-chip`

### `web/index.html`

尽量少改 DOM，但允许做这两类整理：

1. 合并视觉卡片内重复或冗余的 KPI 容器
2. 调整顶部摘要内元素顺序，让状态更清晰

原则：

- 不改 id
- 不改数据绑定
- 不破坏现有 JS 取元素逻辑

## 明确禁止

Claude Code 不要做这些事：

- 不要碰扫码、PLC、相机、串口、接口逻辑
- 不要顺手改设置页
- 不要把生产页改成后台表单风格
- 不要加花哨动画
- 不要引入新框架
- 不要把拖拽/缩放功能删掉

## 验收标准

改完后至少满足下面几点：

1. 第一眼能看出 O 型圈视觉是主区域。
2. 最近记录形成稳定的右侧栏，而不是普通表格块。
3. 左侧三张卡片看起来属于同一组辅助工具。
4. 顶部摘要不再臃肿，也不再像第二套主界面。
5. 页面整体明显更整洁，卡片之间的秩序更强。
6. 默认布局无需手动拖拽，也已经是顺眼的。
