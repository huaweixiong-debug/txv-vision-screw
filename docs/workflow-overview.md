# 膨胀阀拧紧防错追溯 HMI — 完整流程说明

## 系统架构

| 组件 | 型号/地址 | 协议 |
|------|----------|------|
| 相机 | 海康 MV-CU060-10GM @ 192.168.0.101 | MVS SDK (GigE) |
| PLC | 西门子 S7-200 SMART @ 192.168.0.10 | Snap7 (TCP:102) |
| 拧紧枪 | 奇力速 KL-NTCS-M7 @ 192.168.0.105 | Modbus TCP:502 |
| 扫码枪 | Honeywell Vuquest 3310 @ COM3 | USB Serial (CDC) |
| YOLO 模型 | best.pt (D:\ultralytics-main\best.pt) | Ultralytics |
| Web HMI | http://192.168.0.99:8010 | HTTP |

## M 位通讯

### PC → PLC（上位机输出）

| 地址 | 含义 | 触发条件 |
|------|------|----------|
| M0.0 | 产品就绪 | TXV 或 QR 检测通过后置 1，同时清空拧紧旧数据 |
| M0.1 | 拧紧合格 | 两颗螺栓都 OK 后置 1 |
| M0.2 | 扫码完成 | 二维码绑定后置 1 |
| M0.7 | 屏蔽扫码 | 操作员选择跳过扫码 |
| M1.0 | 拧紧不合格 | 任意螺栓 NG 时置 1 |

### PLC → PC（PLC 输入）

| 地址 | 含义 | 说明 |
|------|------|------|
| M0.3 | 手动/自动 | 1=手动，0=自动（唯一判断标准） |
| M0.4 | 急停 | 1=急停激活 |
| M0.5 | PLC 就绪 | 1s 脉冲，1s 间隔（含 1.5s 锁存） |
| M0.6 | PLC 复位 | 复位整个流程 |
| M10.2 | 拧紧完成 | PLC 通知拧紧结束 |

## 自动流程（7 步）

- 自动化启动：M0.3=0 时，前端自动调用 `POST /api/automation/start`
- 进入拧紧前所有拧紧数据清空

### 步骤 1：O 型圈检测

- 画面中出现 O 型圈
- **只要检测到** 2 个不重叠的 O_Ring + 0 个 NG（不需要等待）
- O 型圈步骤变绿色，进入下一步

### 步骤 2：膨胀阀/二维码检测

- 放置膨胀阀遮住 O 型圈
- **只要检测到** TXV 或 QR（任一即可）
- 步骤变绿色

### 步骤 3：压紧（PLC 握手）

- PC 向 PLC 发送 M0.0=1（产品就绪），同时清空奇力速旧数据
- 等待 PLC 发送 M0.5=1（就绪脉冲）
- 收到脉冲后步骤变绿色

### 步骤 4：拧紧

- 进入时清空螺栓状态 + 奇力速基线拍照
- PLC 控制拧紧枪打两颗螺栓
- HMI 每 300ms 轮询 Modbus 读取扭矩和角度
- 步骤变绿色

### 步骤 5：判定

- 两颗螺栓都 OK → M0.1=1 → 进入扫码
- 任意螺栓 NG → M1.0=1 → 等待 PLC 返修选择

### 步骤 6：扫码

- 扫码枪自动触发（HEX: 16 54 0D，每 2s）
- 重码或规则不匹配 → 自动重触发
- 可选择跳过扫码（M0.7=1）
- 步骤变绿色

### 步骤 7：完成

- M0.2=1
- 全部 7 步变绿色
- 记录保存到数据库
- 返回空闲，等待下一个产品

## YOLO 推理逻辑

### 检测类别（4 类显示，5 类模型）

| 模型输出 | 显示为 | 颜色 | 框上限 |
|----------|--------|------|--------|
| NG | NG | 红色 | 2 |
| O_Ring_L | O_Ring | 绿色 | 2 |
| O_Ring_S | O_Ring | 绿色 | 2 |
| QR | QR | 黄色 | 1 |
| TXV | TXV | 绿色 | 1 |

### 去重规则

- O_Ring_L 和 O_Ring_S 统一映射为 O_Ring
- 同类重叠 >70%：保留置信度最高的
- NG 与 O_Ring 重叠：保留置信度最高的
- O_Ring ≥2 个：移除所有 TXV
- TXV 出现：移除所有 O_Ring
- 每类最多保留：NG=2, O_Ring=2, QR=1, TXV=1

### O 型圈合格判定

- 2 个不重叠 O_Ring + 0 个 NG = 合格

### 推理节拍

- 相机抓帧：1000ms（1 FPS）
- 推理间隔：1000ms
- 推理尺寸：416×416
- 置信度：0.15，IoU：0.3
- 前端轮询：1000ms

## 拧紧判定

- 扭矩范围由产品型号决定（如 HB11: 4.0-5.0 N·m）
- 两颗螺栓独立判定
- result_code: 4=OK, 5=OK-SEQ, 6=OK-JOB, 7=NG, 8=NS
- OK = code in (4, 5, 6), NG = code in (7, 8)
- 全部 OK → 合格，任意 NG → 不合格

## 页面操作

### 生产页面

- 顶部：当前步骤 + 步进条 + 模式徽章
- 左上：产品与操作员选择
- 中左：PLC 条件（M 位实时显示）
- 中上：O 型圈视觉（实时画面 + 检测结果列表）
- 右上：奇力速拧紧（扭矩 + 角度 + 结果）
- 中右：扫码区
- 下方：当日记录 + 统计

### 设置页面

- 硬件设置：相机、PLC、自动化参数
- 生产设置：产品型号参数（扭矩上下限、角度上下限、二维码规则）、人员管理

### 训练数据页面

- 左侧：相机实时画面 + 画面控制（缩放/旋转/平移）+ 曝光调节 + 抓拍按钮
- 右侧：生成 YOLO 数据集

### 数据查询页面

- 查询历史记录
- 导出当日 Excel 报表

## 自动启动条件

- 唯一依据：PLC M0.3 = 0（自动模式）
- 不依赖 `settings.automation.enabled`
- HMI 检测到后自动调用 `POST /api/automation/start`
- 无需手动操作

## 页面刷新行为

- 刷新页面不创建新记录
- 记录只在 7 步完成后生成
- 关闭浏览器自动断开所有设备连接

## 产品切换

- 切换产品型号 → 立即更新顶部产品/配方显示
- 同时写入奇力速控制器参数
